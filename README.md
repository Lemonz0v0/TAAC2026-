# TAAC2026-初赛（已开源）
研0小白 研究方向非推荐算法 记录与分享TAAC经验 打这个比赛是因为老板接了个腾讯项目被叫来参加 单打独斗。本人的成绩最开始从600+一路慢慢爬升到最后擦线进复赛。最后提交：val 0.8705，test：0.8235。如果你是小白，可以看看我的心得，顺便点点star！

baseline很强，初期试探了一些方向例如正则、scaling up均无效，学习成本和试错成本都很大。初赛数据集只有100M，加上官方baseline没有完全固定seed，导致同一份代码run多次再test会出现不同的test auc，亲测浮动大概在千3左右（一次0.8194一次0.8167）。有尝试完全固定seed，但是掉了千1多，随放弃固定seed。因此很多改动掉分不一定是不work，有可能是代码浮动，也有可能是不适配现有模型，几乎每次提交都是抽卡，luck is all u need！

平台也是无话可说，莫名其妙的OOM，无论是train还是test，OOM不一定是代码OOM，可能是平台抽风，资源分配bug或抽到矿卡了，再run一次试试看！

涨分的改动基本上都是按照大佬们的思路，一方面是从数据和baseline出发进行补数据，补模型看不到、需要的数据流；另一方面是分析是否梯度重合，有些改动所涉及的方向可能是相同的，如果之前的改动吃到红利了再来一个改动一般做不到1+1>1。所有的涨分改动我没具体记录，只能凭借着时间线和大概涨分进行记录：

1、amp+compile：最开始是为了加速代码，但涨了千1，遂保留。看别的选手的帖子，后期去掉加速反而掉分。可能官方数据集太小，模糊一点精度反而更好。

2、user时间特征：发生在星期几、发生在一天中的第几个小时

3、seq周期时间特征：每个历史行为 timestamp 加东八区偏移后，转成小时桶和星期桶

4、seq-seq pair特征：相邻行为之间的时间差

5、将skip_emp阈值从100w拉到了110w，为了吃seq_c34这个数据，涨了千1左右。

6、token重分布：512→422，给item补一个token，涨了千1左右

7、新增token：由于seq截断，额外保留 raw length、overflow length、是否溢出、保留比例、溢出比例这 5 个统计量，涨了万6

8、时间桶边界修改：把边界重新设计成更细、更长尾的分布，涨了千1.5左右

9、DIN-like 的 item-to-sequence attention，涨了千1多

10、dense重分组：将 61, 89, 90 放到一个组，将 62, 63, 64, 65, 66, 87, 91 放到另一个组，涨了千1

11、dense-only EMA：0.999，涨了千1左右，0.998、0.997效果不好

12、label smoothing：涨了万3，最后一天进复赛的绝杀上分

13、（不确定）lr warm up：这个也是凭直觉的，最开始为了避免学到噪声、无用信息，所以采用dense-only lr warm up，但是掉了千1，不知道为什么。我认为是代码波动，但一直保留，最后忘了做消融

print训练集和测试集发现，测试集时间线是在训练集之后的，训练集大概持续3天，测试集只有3小时，在某一天的23、24、01点，从数据集出发我认为需要关注的一点就是ood。但前期凭直觉做了一些正则例如EMA、label smoothing、增大dropout均无效，可能是数据集太小，模型本身就学不到什么再正则一下更学不到了。另一个从数据集发现的问题是，baseline的4个seq均被截断很多，尤其是seq_d被截断了90%，从直觉上来说扩大seq补充信息可以让模型更好的学习，但是我在初期和后期均尝试了单一扩大和全部扩大seq消融实验，均掉分，我怀疑是seq的噪声过大，但没一直没想出什么去噪方法。还有一个尝试：baseline是将缺失值和本身就为0的值均赋值为0，我尝试将其分离，把本身就为0的数据加入学习，剔除掉缺失值，但也掉分。由于扩大seq长度会掉分，所以考虑另辟蹊径的方法，将截断多少等信息喂给模型（上述7）；除此之外print训练集和测试集还有重要发现，就是数据之间的相关性，以此依据做了上述10决定

我的baseline val0.8620，test0.8123。我看到比赛期间有人讨论关于epoch提交问题，有选手消融发现提交非best val模型和非best logloss模型反而test更高（千2左右）。我认为和个人的模型有关，我最开始从baseline上面改，best val均在epoch6，实测epoch4、5比不过epoch6。当加入EMA改动之后best val出现在了epoch5，epoch4和epoch6也比不过epoch5。比赛期间test次数有限，所以当best val的epoch未发生偏移的时候，做一次消融就足够了，当best val的epoch发生偏移的时候，需要再做消融。

还有一些尝试，例如从一开始就听说优化器可以上分，但是我无论是替换优化器、优化器调参还是像LLM一样layer层等不走优化器，都是掉分；还有将baseline的rankmixer替换成unimixer、tokenmixer、tokenformer也掉分，但有选手涨分，可能不适配我的模型。
